# 1. Purpose

This document defines the official architecture scope of Art of Business v1.0.

It establishes what is included in the canonical architecture release, what is excluded from the release boundary, and what becomes part of future productization, implementation, or research branches.

---

# 2. Scope Declaration

Art of Business v1.0 covers the complete conceptual and operational architecture of an AI-orchestrated enterprise system through the Global Intelligent Economy level.

```text
Release Scope: Architecture Baseline
Release Version: v1.0
Release Status: Frozen
Implementation Status: Approved for Productization
```

# 4. In-Scope Layer Range

The canonical scope includes architectural layers up to:

```text
23_GLOBAL_INTELLIGENT_ECONOMY
```

This means that all architecture layers from the foundation through Layer 23 are treated as part of the v1.0 canonical baseline.

---

# 5. Out-of-Scope for v1.0

The following items are outside the v1.0 architecture release scope:

- production software implementation;
- runtime code;
- cloud deployment scripts;
- working user interface;
- commercial pricing model;
- customer onboarding playbooks;
- marketplace implementation;
- SDK implementation;
- legal entity structure;
- investor materials;
- sales collateral.

These items belong to the productization branch.

---

# 6. Productization Boundary

The approved implementation branch after v1.0 is:

```text
24_PRODUCTIZATION_AND_IMPLEMENTATION
```

This branch transforms the canonical architecture into the Bizzi Platform reference implementation.

# 7. Research Boundary

The optional theoretical research branch after v1.0 is:

```text
24_CIVILIZATION_OPERATING_SYSTEM
```

This branch is explicitly separated from the productization branch and should not block implementation of Bizzi Platform.

---

# 8. Architecture vs Product

Art of Business v1.0 defines:

```text
What the system should be
Why it exists
How its governance works
How agents, processes, memory, autonomy, risk and evolution are structured
```

Bizzi Platform defines:

```text
How the system is implemented
How users interact with it
How agents run
How tools connect
How data is stored
How the product is deployed and sold
```

---

# 9. Scope Inclusion Criteria

A document is included in Art of Business v1.0 if it defines one or more of the following:

- architectural principle;
- governance rule;
- operating model;
- agent role;
- enterprise function;
- process model;
- risk model;
- memory model;
- autonomy model;
- evolution model;
- ecosystem model;
- network economy model;
- global intelligent economy model.

---

# 10. Scope Exclusion Criteria

A document is excluded from Art of Business v1.0 if it primarily defines:

- source code;
- infrastructure-as-code;
- deployment scripts;
- implementation backlog;
- UX wireframes;
- product pricing;
- marketing assets;
- customer contracts;
- operational runbooks for a specific company;
- vendor-specific technical configuration.

---

# 11. Canonical Baseline Rule

After v1.0 release, canonical architecture changes must be versioned.

Allowed paths:

```text
v1.0.x — patch corrections
v1.x — compatible architecture extensions
v2.0 — major architecture revision
Bizzi Platform — implementation branch
Research Architecture — experimental architecture branch
```

---

# 12. Scope Stability

The v1.0 scope is stable and should not be expanded casually.

New architectural concepts may be added only when they are:

- clearly outside existing layers;
- approved as part of a new release;
- mapped to existing governance and risk models;
- documented with traceability;
- reviewed by audit.

---

# 13. Implementation Readiness Boundary

The architecture is ready for implementation, but implementation artifacts are not part of this release.
