# 6. Core Artifact Families

## 6.1 Governance Artifacts

Define authority, accountability, decision rights, escalation, policy and audit obligations.

## 6.2 Capability Artifacts

Define enterprise capabilities and capability decomposition.

## 6.3 Function Artifacts

Define enterprise functions, owners and responsibility boundaries.

## 6.4 Agent Artifacts

Define AI agents, roles, authority boundaries, responsibilities and audit requirements.

## 6.5 Process Artifacts

Define business processes, lifecycle, ownership and execution dependencies.

## 6.6 Risk and Compliance Artifacts

Define risk domains, controls, compliance obligations, escalation and mitigation.

## 6.7 Operations Artifacts

Define operational execution, operating rhythms, performance visibility and coordination.

## 6.8 Intelligence Artifacts

Define observability, metrics, analytics, audit intelligence, dashboards and knowledge graph.

## 6.9 Memory Artifacts

Define knowledge memory, decision memory, process memory, agent memory, context and lessons learned.

## 6.10 Autonomy Artifacts

Define governed autonomy, self-healing, self-optimization, self-planning and autonomous execution.

## 6.11 Evolution Artifacts

Define learning, adaptation, self-improvement, process evolution, agent evolution and strategic evolution.

## 6.12 Ecosystem Artifacts

Define partner, supplier, customer, multi-enterprise and ecosystem governance models.

## 6.13 Networked Economy Artifacts

Define market networks, platform economy, value networks, economic orchestration and network governance.

## 6.14 Global Intelligent Economy Artifacts

Define global economic intelligence, global coordination, platform infrastructure, AI economic agents, value flows, risk, governance and evolution.

# 9. Artifact Inclusion Rule

An artifact belongs to the v1.0 canonical baseline if it defines architecture rather than implementation.

Included artifact types:

- architecture principles;
- governance models;
- function definitions;
- process models;
- agent definitions;
- risk models;
- compliance models;
- memory models;
- autonomy models;
- ecosystem models;
- economy models;
- release-control documents.

---

# 10. Artifact Exclusion Rule

The following artifact types are excluded from the v1.0 canonical architecture baseline:

- source code;
- infrastructure scripts;
- deployment templates;
- UI wireframes;
- product pricing;
- sales materials;
- customer onboarding playbooks;
- vendor-specific runtime configuration;
- implementation tickets.

These belong to the productization branch.
