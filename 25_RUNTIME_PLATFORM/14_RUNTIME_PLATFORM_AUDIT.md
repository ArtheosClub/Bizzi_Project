# 22. Identified Risks

## Risk 1 — Runtime Breadth

The runtime scope is broad for an MVP.

Mitigation:

```text
Build the first runnable vertical slice before implementing the full runtime breadth.
```

## Risk 2 — Premature Complexity

The runtime could become over-engineered.

Mitigation:

```text
Use modular monolith, internal events and simple service boundaries first.
```

## Risk 3 — AI Governance Drift

AI could be implemented too autonomously if engineering shortcuts are taken.

Mitigation:

```text
Preserve the MVP rule: AI recommends, human confirms.
```

## Risk 4 — Weak Source Traceability

Memory and audit may lose value if source links are not enforced.

Mitigation:

```text
Require workspace_id, source_object_type and source_object_id where applicable.
```

## Risk 5 — Empty Product Experience

Runtime objects may exist but dashboard may not show useful value.

Mitigation:

```text
Use onboarding to generate operating map, tasks, gaps, memory and dashboard indicators early.
```
